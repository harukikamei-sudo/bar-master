# 要件定義書：3Dバーカウンター AIマスターお酒レコメンドアプリ

## 1. プロジェクト概要

### 1.1 プロジェクト名
BAR MASTER - 3Dバーカウンター お酒レコメンドWebアプリ

### 1.2 コンセプト
3Dで再現されたバーカウンターに座り、バーテンダー（マスター）と会話しながら自分に合ったお酒を見つけられるWebアプリケーション。ゲームのショップ画面のようなUI/UXで、マスターが提案したお酒の3Dモデルが目の前にポップアップし、詳細情報を確認しながら選べる。

### 1.3 ターゲットユーザー
- お酒に興味はあるが詳しくない初心者
- 新しいお酒を開拓したい愛好家
- バーに行く前に予習したい人
- ECサイトでのお酒購入を検討している人

---

## 2. 画面構成・UI設計

### 2.1 レイヤー構造

**背景レイヤー（3Dシーン / Blenderで制作）**
- バーカウンター（静的・客席視点）
- バックバー棚（静的・マスターの背後）
- バーテンダー（マスター）3Dキャラクター（idle/bow/talk/thankアニメーション）
- 暖色系の環境ライティング

**前面レイヤー（ゲームUI）**
- 会話ウィンドウ（画面下部）
- お酒詳細パネル（画面中央にポップアップ）
- 選択済みリスト（サイドバー）
- ハンバーガーメニュー（法的ページ）

### 2.2 お酒詳細パネル（メインUI）
マスターがお酒を提案した際に画面中央にポップアップ表示。ゲームのアイテム詳細画面のようなデザイン。

**表示要素：**
- お酒の3Dモデル（自動回転表示）
- 商品名
- 種類（シャンパン/ウイスキー/コニャック/ワイン等）
- 説明文（マスターの言葉で）
- 味わいプロフィール（甘味・苦味・酸味・ボディ・香り をレーダーチャートで）
- アルコール度数
- 価格
- おすすめのつまみ・ペアリング
- 「これにする」ボタン
- 「他のも見たい」ボタン
- 「違うジャンルで」ボタン

### 2.3 会話ウィンドウ
- 画面下部に配置
- マスターの発言（左側・アイコン付き）
- ユーザーの選択（右側）
- 選択肢ボタン＋フリーテキスト入力による会話進行
- タイプライター風テキスト表示（40ms/文字）

### 2.4 選択済みリスト
- 会話中に「これにする」と選んだお酒の一覧
- 名前 + 価格
- 削除ボタン
- 合計金額表示

## 3. 機能要件

### 3.1 会話機能（事前定義型 + キーワードマッチ）

| 項目 | 詳細 |
|------|------|
| 方式 | キーワードマッチ（30+パターン）+ 選択肢ボタン + フリーテキスト入力（API不使用） |
| 人格 | 40代の経験豊富な日本のバーテンダー。穏やかで聞き上手、知識豊富だが押し付けない |
| 会話フロー | 気分・好みのヒアリング → 条件に合ったお酒を提案 → 詳細説明 → 決定 or 次の提案 |
| 分岐ロジック | キーワードマッチによるフィルター型 |
| 提案トリガー | マスターがお酒を提案した際、タイプライター完了後に詳細パネルをポップアップ |

### 3.2 会話シナリオ構成

**ステップ1：来店**
- ローディング画面 → 「Enter」ボタンで入店
- マスター：時間帯別挨拶（朝昼/夕方/夜/深夜）
- 再訪検知（localStorage）で挨拶が変化

**ステップ2：好みの味ヒアリング**
- 選択肢：「甘めが好き」「辛口・ドライ」「フルーティ」「苦味があるもの」「おまかせ」

**ステップ3：提案**
- マスター：選択肢の組み合わせに応じたお酒を提案 → 詳細パネルポップアップ

**ステップ4：決定 or 継続**
- 選択肢：「これにする」「他のも見たい」「違うジャンルで」

**常連システム：**
- 訪問5回以上：全品10%OFF（マスターから割引告知）
- 訪問10回以上：割引告知の後、裏メニュー提案（ロマネ・コンティ）

### 3.3 お酒データベース

| 項目 | 詳細 |
|------|------|
| 登録数 | 4銘柄（通常3 + 裏メニュー1） |
| カテゴリ | シャンパン、ウイスキー、コニャック、ワイン（裏メニュー） |
| データ項目 | 名前、種類、説明、味わいパラメータ（甘味/苦味/酸味/ボディ/香り 各1-5）、度数、価格、産地、おすすめペアリング、3Dモデルファイルパス、タグ（気分・シーン紐付け） |
| フィルタリング | 気分タグ × 味の好み の組み合わせでスコアリングマッチング |
| 拡張性 | JSONデータ追加で銘柄追加可能 |

### 3.4 3Dシーン機能

| 項目 | 詳細 |
|------|------|
| エンジン | Three.js v0.183（React Three Fiber + drei） |
| モデル形式 | glTF 2.0（.glb）/ Draco圧縮 |
| 制作ツール | Hitem3D → Blender → エクスポート |
| 背景シーン | バーカウンター + バックバー棚（bar_scene.glb 5.1MB） |
| マスター | master.glb（4.8MB）idle/bow/talk/thank 4アニメーション、crossFade 0.4s |
| 商品3Dモデル | 詳細パネル内で自動回転表示 |
| ライティング | 暖色ポイントライト + 環境光（バーの雰囲気） |
| カメラ | 入場演出（X軸スライド 5→0.614、easeOutCubic）→ 固定視点 |

### 3.5 UI演出

| 項目 | 詳細 |
|------|------|
| ローディング | ゴールドバー進捗表示 → 「Enter」ボタンで入店 |
| パネルポップアップ | マスター提案時、タイプライター完了後にスムーズ表示 |
| ボトル登場演出 | 3Dモデルが自動回転しながら表示 |
| 会話テキスト | タイプライター風に1文字ずつ表示（40ms/文字） |
| BGM | テーマ曲ループ再生（音量30%、ミュートボタン付き） |
| 効果音 | グラス音（注文時）、ベル音（お会計時） |

## 4. 技術構成

### 4.1 フロントエンド

| 技術 | 用途 |
|------|------|
| Vite v8 | ビルドツール |
| React v19 | UIコンポーネント管理 |
| Three.js v0.183（R3F + drei） | 3Dシーン描画 |
| Tailwind CSS v4 | UIスタイリング |
| CSS | カスタムアニメーション・テーマ |

### 4.2 データ管理

| 技術 | 用途 |
|------|------|
| JSON | お酒データベース（アプリ内定数） |
| JS（chatPatterns.js） | キーワードマッチ応答パターン（30+） |
| React State | 会話状態・選択履歴管理 |
| localStorage | 訪問回数・前回購入記録（常連システム） |

※ 外部API不使用。すべてフロントエンド完結。

### 4.3 3Dアセット制作パイプライン

| 工程 | ツール | 詳細 |
|------|--------|------|
| モデル生成 | Hitem3D | 高品質3Dモデル生成 |
| 調整・組立 | Blender | スケール合わせ、マテリアル調整、シーン組立、キャラクター配置 |
| アニメーション | Mixamo + Python | Mixamoで取得 → bakeスクリプトで抽出 → injectスクリプトでglbに注入 |
| エクスポート | Blender | glTF 2.0（.glb）形式、Draco圧縮 |

## 6. 画面遷移フロー

```
[ローディング画面] 3Dシーン読み込み → ゴールドバー進捗
    ↓ 「Enter」ボタンクリックで入店
[バーカウンター画面] カメラ入場演出 → マスターが挨拶
    ↓ 選択肢 or テキスト入力で回答
[好みヒアリング] 味の好みを質問
    ↓ 選択肢 or テキスト入力で回答
[お酒提案] マスターが提案 → タイプライター完了後 → 詳細パネルポップアップ
    ↓ 「これにする」or「他のも見たい」or「違うジャンルで」
[決定 / 次の提案] 選択済みリストに追加 or 別の提案へ
    ↓ 「お会計」
[結果画面] 選んだお酒一覧・合計表示
```
